REQUISITOS DE OO

Atende aos requisitos de OO da primeira entrega?
R: 
Tem alta coesão?
R: Sim. Cada classe tem uma responsabilidade clara e bem definida.
Tem baixo acoplamento?
R: Sim. As classes interagem de forma adequada, com dependências mínimas entre si.
Usa design baseado em responsabilidade?
R: Sim. Cada classe é responsável por uma funcionalidade específica
Tem acoplamento implícito?
R: Não. As dependências entre as classes são explícitas e bem definidas.
Tem replicação de código?
R: Não. Usamos herança e métodos compartilhados para contornar esse problema.
Tem duas superclasses com subclasses com implementação útil?
R: Sim. As superclasses Botao (subclasses: BotaoProximaFase, BotaoReiniciarJogo, BotaoVoltarMenu) e Fruta (subclasses: Maca, MacaDourada, MacaPodre)

COMENTÁRIOS (OPCIONAIS)

Algum comentário sobre o tema do trabalho?
Sem dúvidas, foi muito interessante e desafiador para nós criar um jogo do zero. Como os requisitos foram
definidos de uma maneira clara, modelar o jogo foi bem se tornou uma tarefa bem mais tranquila.

Algum comentário sobre o Greenfoot?
No início tivemos resalvas, tendo em vista que estavamos com problemas com um áudio específico, porém, descobrimos que era um áudio corrompido.
No fim das contas, é um grande facilitador, sobretudo porque o controle e modificação das classes se torna muito fácil com ele.
